Search in VR

UX Exploration

Today, text input in VR is a challenging task. This exploration aims to test out new UX concepts to improve text input.

The Problem

Text input is important in a number of scenarios such as searching for items, inputting pins, etc. When we start up our VR experience, one of our first tasks is to find an app to launch either by browsing or searching. I was inspired to start this project when I noticed how much I struggled to search for apps in VR, and how unnatural it felt to input text. Is there a way to input text for a search in a more natural and comfortable way? That's the problem I'm trying to solve.

To start, how does text input work in VR? Most VR apps provide a giant digital QWERTY keyboard to input text, which makes sense because this is similar to how we input text in desktop and mobile. Users select characters either by

  1. Moving a laser pointer with our wrist/arm
  2. Pushing an analog stick to select (similar to the console solution)
  3. Hovering over a character and clicking a touchpad to select (e.g. SteamVR keyboard)

All three of these solutions work, but why does this feel awkward? A "natural" input solution doesn't require us to "think" about how to communicate the text. Handwriting feels "natural," because we have practiced it since we are children, so there is limited cognitive load for us to write. Typing on a physical keyboard is something many people today have adapted, and it is well designed for fast and accurate input, especially for languages with ASCII based character, so that feels pretty good

The SteamVR keyboard is an example of a VR keyboard that feels very unnatural. The keyboard is split in half with the left side of the keyboard controlled by the left touchpad, and the right by the right touchpad. It's awkward because this is a new way of input to many of us and it's a bit confusing. We don't frequently interact with a split keyboard and it's not clear why each touchpad can only interact with a set amount of keys. Yes the iOS keyboard can be split, but it is not the default state and there is a physical separation between the keyboard itself that makes sense for the keyboard to be split visually.

I can imagine users acclimating to the SteamVR keyboard to a degree at some point. Is there a more natural form of input for text though? For this exercise, I scoped the problem to text input for search queries.

The Idea

What if users can write their search? We've been writing since we were kids, so we are very familiar with this form of input, which makes it easier to pick up. Handwriting certainly won't win in any speed competition, but with autocomplete smartness the length of search queries doesn't have to be very long.

To test this idea I built a basic prototype with Unity. The scene is set up so that you can search through a store catalog. By writing letters on a whiteboard, I run OCR and use that as your input query, and return results. I had a few users test this prototype and they agreed that this form of input felt more "natural" than the digital keyboard they have experienced on VR so far. However, there were times where the OCR would not recognize the writing correctly and rewriting the query can be frustrating.

If you would like to try the prototype, you can find the code here . Please note this is a prototype! There are a number of cases that the prototype does not account for (e.g. if you are taller than I am, the whiteboard may seem low.). Also the OCR results would be improved if the writing was written with lines instead of cubes.




Final Thoughts

Using handwriting for search input can feel more natural. The biggest area of concern is how accurate the OCR results are. With keyboard input users will also make mistakes so the question is is OCR good enough to be acceptable for users.

For next steps, I want more users to test this prototype out and continue to collect more feedback. In addition, I plan on exploring other UX concepts such as voice input or other types of keyboards. I think there is room for improvement.
